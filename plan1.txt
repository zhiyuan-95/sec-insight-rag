Milestone 1 Design: Project Scaffold

Task Restatement
Build the initial backend scaffold for the financial research assistant. This part should establish the repository structure, dependency management, configuration loading, a minimal FastAPI entrypoint, basic tests, and local run instructions. It should not implement SEC ingestion, XBRL normalization, financial indicators, retrieval, Gemini integration, or database schema yet.

Success Criteria
- The repository has the planned backend folder structure from proposal.md.
- Dependencies are managed with pyproject.toml and uv.
- Configuration can be loaded from config.env without hardcoding secrets or local paths in source code.
- A minimal FastAPI application starts successfully.
- GET /health returns a simple healthy response.
- Unit tests validate configuration loading and the health route.
- Local setup, run, and test commands are documented.

Important Edge Cases
- config.env keys may include spaces around names or values, such as "Gemini_API_KEY = value".
- Some proposal variables may be missing in the current config.env, especially KNOWLEDGE_STORAGE_DIR.
- Secret values must not be logged, printed, or copied into documentation examples.
- The app should start without requiring a live SEC request or Gemini API call.
- Missing SEC_USER_AGENT should not block Milestone 1 startup, but the settings model should expose the value for future ingestion validation.
- Storage paths should be configurable and resolved consistently, but directories should not be populated with generated data in this milestone.

Dependency Design
Use uv with pyproject.toml.

Runtime dependencies:
- fastapi
- uvicorn
- pydantic-settings
- python-dotenv

Development dependencies:
- pytest
- httpx

Use Python 3.11 or newer unless the local machine already requires a different Python version.

Configuration Design
Create src/config/settings.py with a typed Settings model and one public loader:

- Settings
  - gemini_api_key: optional secret string loaded from Gemini_API_KEY
  - sec_user_agent: optional string loaded from SEC_USER_AGENT
  - stock_sql_db_path: path loaded from STOCK_SQL_DB_PATH, defaulting to stock_data.db for local development
  - stock_storage_base_dir: path loaded from STOCK_STORAGE_BASE_DIR, defaulting to ./storage/stock
  - stock_filings_base_dir: path loaded from STOCK_FILINGS_BASE_DIR, defaulting to ./data_store/filings
  - knowledge_storage_dir: optional path loaded from KNOWLEDGE_STORAGE_DIR
  - primary_chat_model: string defaulting to gemini-2.5-flash
  - allowed_chat_models: list defaulting to ["gemini-2.5-flash"]

- load_settings(env_file: str | Path = "config.env") -> Settings
  - Loads the local env file.
  - Trims whitespace around keys and values.
  - Accepts the existing mixed-case Gemini_API_KEY name.
  - Does not print loaded values.

Do not reject missing Gemini_API_KEY in Milestone 1 because no LLM call is implemented yet.
Do not reject missing SEC_USER_AGENT in Milestone 1 because no SEC call is implemented yet.

API Design
Create src/api/main.py with:

- create_app() -> FastAPI
  - Builds and returns the FastAPI application.
  - Registers the health route.

- app
  - Module-level app instance for uvicorn.

Route:
- GET /health
  - Response: {"status": "ok"}
  - No external service checks in Milestone 1.

Prompt Design
Create src/analyze/prompts.py as the future single source of truth for prompt templates.
For Milestone 1, keep it minimal. It can contain only a module docstring explaining that production prompt templates must live there. Do not define inline prompts in routes or tests.

Storage and Data Design
Create placeholder directories only:
- data/fixtures for immutable saved SEC responses and filing samples.
- data/exports for generated CSV exports later.

Do not create database tables.
Do not write sample generated data.
Do not modify raw data files.

Testing Design
Add tests for configuration loading:
- Loads values from a temporary env file.
- Trims whitespace in env keys and values.
- Keeps missing KNOWLEDGE_STORAGE_DIR valid.
- Confirms primary_chat_model defaults to gemini-2.5-flash.
- Confirms allowed_chat_models only includes gemini-2.5-flash by default.

Add tests for the API scaffold:
- create_app returns a FastAPI app.
- GET /health returns status code 200.
- GET /health returns {"status": "ok"}.

Commands to verify:
- uv sync
- uv run pytest
- uv run uvicorn src.api.main:app --reload

Documentation Design
Add local setup instructions in README.md or docs/local_run.md:
- Install uv if needed.
- Run uv sync.
- Run tests with uv run pytest.
- Start the backend with uv run uvicorn src.api.main:app --reload.
- Note that config.env is local and may contain secrets.
- Note that Milestone 1 does not contact SEC, Gemini, or any external service.

Out of Scope
- SEC ticker-to-CIK lookup.
- SEC submissions retrieval.
- Companyfacts retrieval.
- Filing downloads.
- XBRL normalization.
- SQLite schema creation.
- Derived indicators.
- Deterministic analytics.
- LlamaIndex retrieval.
- Gemini model calls.
- RAG analysis.
- CSV export routes.
- Any frontend code.

Implementation Order
1. Add pyproject.toml with uv-compatible project metadata and dependencies.
2. Add backend package folders and __init__.py files.
3. Add settings loader in src/config/settings.py.
4. Add FastAPI app and health route in src/api/main.py.
5. Add prompt placeholder module in src/analyze/prompts.py.
6. Add test files for settings and health route.
7. Add local run documentation.
8. Run uv sync, then uv run pytest.

Assumptions
- Part 1 means Milestone 1, "Project scaffold", from proposal.md.
- The selected dependency approach is pyproject.toml with uv.
- The implementation should make the smallest useful scaffold and avoid premature architecture beyond the proposal's folder structure.
- Future milestones will add SEC ingestion, storage schema, analytics, retrieval, Gemini integration, and API routes.
